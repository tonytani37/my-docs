# mkdocsをつかってgithub pagesからドキュメントを公開する方法

# 最初に
ローカルでmkdocsが動くかどうか確認する

# mkdocsのインストール方法

## 💻 ステップ 1: MkDocsのインストール

Pythonのパッケージ管理システムである`pip`を使って、MkDocsとその依存関係をインストールします。

1.  \*\*ターミナル（コマンドプロンプトやPowerShellなど）\*\*を開きます。
2.  以下のコマンドを実行します。
    ```bash
    pip install mkdocs
    ```
    これにより、MkDocsの実行に必要なすべてのファイルがローカルPCに導入されます。

-----

## 🚀 ステップ 2: プロジェクトの作成

新しいMkDocsプロジェクトを初期化し、基本的なファイルを作成します。

1.  **プロジェクトを作成したいディレクトリに移動します。**
    ```bash
    mkdir my-local-mkdocs-project
    cd my-local-mkdocs-project
    ```
2.  **プロジェクトを初期化します。**
    ```bash
    mkdocs new .
    ```
    このコマンドを実行すると、現在のディレクトリ（`.`）に以下のファイルが生成されます。
      * `mkdocs.yml` (設定ファイル)
      * `docs/` フォルダ
          * `docs/index.md` (最初のドキュメントファイル)

-----

## 🔎 ステップ 3: ローカルサーバーの起動と確認

作成したプロジェクトをローカルサーバーで起動し、ブラウザでプレビューします。

1.  **プロジェクトのルートディレクトリ**（`mkdocs.yml`がある場所）で、以下のコマンドを実行します。
    ```bash
    mkdocs serve
    ```
2.  サーバーが起動すると、ターミナルに以下のようなメッセージが表示されます。
    ```
    INFO    - Serving on http://127.0.0.1:8000
    ```
3.  ブラウザを開き、表示されたURL **`http://127.0.0.1:8000`** にアクセスします。

これで、ローカルPCにインストールされたMkDocsを使って、ウェブサイトが起動し、内容を確認できるようになりました。

> 💡 **ポイント:** サーバー実行中に`docs/`フォルダ内のファイルを編集・保存すると、ブラウザが自動的に更新されます（ライブリロード機能）。

サーバーを停止したい場合は、ターミナルで `Ctrl + C` を押してください。

# githubにローカルの資源をプッシュする

この時点でローカルとgithub間で認証が完了しているので、通信できる。これはVisual Studio Codeでやってても同じ。

そのフォルダからコマンドを入力すると、公開まで一気に処理が行われる。

```
mkdocs gh-deploy
```
これで動かないときには
```
python -m mkdocs gh-deploy
```
(macOSの場合はpython3とバージョンを明示する必要がある）

現在そのフォルダから関連付けられているリポジトリの確認コマンド

```
git remote -v
origin  https://github.com/tonytani37/my-docs.git (fetch)
origin  https://github.com/tonytani37/my-docs.git (push)
```

こんな感じで表示される。
